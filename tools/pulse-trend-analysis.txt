Type: ES|QL

Description:
Analyses the 4-week trend trajectory for a specific project. Returns how
burn rate, velocity, and sentiment have changed week-over-week — not just
current values but acceleration direction. Use this after pulse-sensor
detects a distress project to add trend context to the signal.

Parameters:
project_id (string, required) — e.g. "LIVE-001"

Query:
FROM projects_live_timeseries
| WHERE project_id == ?project_id
| SORT week ASC
| STATS
    weeks_tracked        = COUNT(*),
    burn_week_earliest   = MIN(budget_burn_rate_wk),
    burn_week_latest     = MAX(budget_burn_rate_wk),
    velocity_earliest    = MAX(velocity_points),
    velocity_latest      = MIN(velocity_points),
    sentiment_earliest   = MAX(sentiment_score),
    sentiment_latest     = MIN(sentiment_score)
| EVAL burn_change_pct     = ROUND(((burn_week_latest - burn_week_earliest) / burn_week_earliest) * 100, 1)
| EVAL velocity_change_pct = ROUND(((velocity_latest - velocity_earliest) / velocity_earliest) * 100, 1)
| EVAL sentiment_change    = ROUND(sentiment_latest - sentiment_earliest, 2)
